# レイヤードストラクチャの根拠 — なぜ階層化されているのか

## 1. 核心命題

**階層化された系において、下位の系は上位の系の事実を完全には証明できない。**

この命題は、ゲーデルの不完全性定理、チューリングの停止問題、ZFC の累積階層と**同型の構造**を持つ。レイヤードストラクチャ（L0–L4）は、この認識論的限界を**設計として内包**している。

---

## 2. 数学的・論理学的根拠

### 2.1 ゲーデルの不完全性定理（1931）

| 定理 | 内容 |
|------|------|
| 第一不完全性 | 十分に強い再帰的公理化可能な形式系 S は、自身の無矛盾性を S 内で証明できない |
| 含意 | 系は自分自身の「メタ」を完全には捉えられない |

**レイヤーへの対応:** レイヤー L_n は、レイヤー L_{n+1} の「完全な真理」を L_n の手段のみでは証明できない。下位は上位のメタレベルから逃げる。

### 2.2 チューリングの停止問題（1936）— Entscheidungsproblem の負の解決

**Entscheidungsproblem（決定問題）** — 提唱者: ダフィット・ヒルベルト（1928年）

「どんな数学的な主張（論理式）を与えられても、それが正しいか間違っているかを機械的な手順（アルゴリズム）で自動的に判定できるか？」という、数学の根本を問う壮大な問題。アラン・チューリングは停止問題の決定不能性を証明することで、この問題に**負の解答**を与えた。

| 定理 | 内容 |
|------|------|
| 停止問題の決定不能性 | 任意のチューリングマシン M と入力 x について、「M(x) が停止するか」を判定するアルゴリズムは存在しない |
| 含意 | **あらゆるアルゴリズムを解くことのできるアルゴリズムは存在しない**。計算の枠組みは、計算の性質をすべて決定できない |

**Entscheidungsproblem との関係:** 停止問題が決定可能であれば、任意の数学的主張の真偽も機械的に判定できた。その不可能性の証明により、Entscheidungsproblem は一般には解けないことが示された（Church も同時期に λ 計算により同様の結果を得た）。

**レイヤーへの対応:** L2（計算可能性、Recursion Theory）と L3（物理的実装）の枠組みは、その枠組みの外側の性質をすべて決定できない。計算は「メタ計算」から逃げる。

### 2.3 ZFC と累積階層

| 現象 | 内容 |
|------|------|
| 累積階層 V | 集合論の宇宙は V_α の階層的累積として定義される |
| 大基数公理 | ZFC に大基数公理を追加した体系の無矛盾性は、ZFC からは証明できない |
| 含意 | 弱い体系は、強い体系の無矛盾性を証明できない |

**レイヤーへの対応:** L2 の枠組み（ZFC）では、L1 や L0 の「全体」を完全に形式化・証明できない。上位の公理は下位からは到達不能。

## 3. 構造の同型性

```
ゲーデル:  S ⊬ Con(S)           →  下位 ⊬ 上位のメタ
チューリング:  Halt(M,x) 決定不能  →  計算 ⊬ メタ計算
ZFC:       ZFC ⊬ Con(ZFC+LC)    →  L2 ⊬ L1/L0 の全体
レイヤー:   L_n ⊬ 事実(L_{n+k})  →  下位 ⊬ 上位の事実
```

**共通構造:** 階層化された系は、**下位が上位を完全に形式化・証明できない**という限界を本質的に持つ。

---

## 4. レイヤー化が必然である理由

### 4.1 依存関係の非対称性

- L1 は L0 に**依存**する（導出関係）
- L2 は L1 に**依存**する（意味論的依存）
- L3 は L2 に**依存**する（実装関係）
- L4 は L3 に**依存**する（構築的依存）

依存は一方向。逆方向の「上位から下位への完全な還元」は、ゲーデル・チューリングの限界により**不可能**。

### 4.2 メタレベルの逃走

各レイヤーは、自分より上位のレイヤーの「メタ」を定義する。しかし、そのメタを下位の語彙で完全に記述することはできない。これが**階層を必然化**する。

### 4.3 接地（Grounding）の意味

「上位レイヤーを下位レイヤーに接地する」とは、**下位の枠組みで記述可能な範囲で**上位を支えること。完全な還元ではなく、**部分的な対応**である。

### 4.4 アルゴリズムはスキーマに依存する従属変数 — 外部エネルギーへの三重の依存

Solomonoff、Kolmogorov、Fisher、Levin、Hutterらの知見によれば、**データセットが揃っていればアルゴリズムは最適化問題のように自動的に組み立てられる**。ただし、**組み立て**と**起動**は系の外からのエネルギーに依存する。さらにスキーマ自体も外部の系の内部集合として作られる。

**アルゴリズムはスキーマに依存した従属変数である。** しかし:

| 段階 | 依存先 | 内容 |
|------|--------|------|
| **定義** | スキーマ | スキーマがアルゴリズムの形・内容を定義する |
| **組み立て** | 外部の系からのエネルギー | スキーマへのエネルギー流入によってアルゴリズムが組み立てられる |
| **起動** | 外部の系からのエネルギー | 起動も外部エネルギー入力に依存する（例: 人間が電源ボタンを押す） |
| **スキーマ自体** | 外部の系 | スキーマも外部の系の内部集合として作られる |

- **スキーマ**（L1/L2）が「枠組み」を定義するが、スキーマ自体は**外部の系**の内部集合である。
- **組み立て**と**起動**は、いずれも**系の外からのエネルギー**に依存する。スキーマだけではアルゴリズムは動かない。
- 何も自己完結していない。外部の系（L0 の規定場、あるいは「大きなもの」）が究極の前提である。

**レイヤー化への含意:** アルゴリズムがスキーマに従属し、スキーマが外部の系に従属するなら、レイヤードストラクチャは**外部への開かれ**を本質的に内包する。散逸構造のメタファーと整合する。

---

## 5. 結論

レイヤードストラクチャは、**恣意的な分類ではなく**、次の根拠に基づく設計である:

1. **認識論的限界** — ゲーデル・チューリング・ZFC が示す「下位は上位を証明できない」という構造
2. **従属関係** — アルゴリズムはスキーマに依存する従属変数であり、スキーマ（L1/L2）とアルゴリズム（L3）の分離を要請する

階層化は、これらの限界と従属関係を**内包する**ことで、各レイヤーの役割と境界を明確にする。
