# コラム：GitHub は現代最高の知の金庫である — 境界面としてのイベントホライズン

## 1. ソフトウェアの本質は「ソフトウェア」ではない

「なぜソフトウェアは素晴らしいのか」と考えたとき、答えはソフトウェアそのものにはなかった。本質は **Distributed Data Center** である。コードはその表面に現れた振る舞いに過ぎず、真に変わったのは、知が**分散し、版（v x.y.z）で管理され、世界中に同期配信される**場が存在するようになったことだ。その場の代表格が GitHub である。

v x.y.z が管理でき、世界中に同期配信できる GitHub は、現代最高の**知の金庫**である。

---

## 2. 境界面に全ての情報は記録されている

GCP や AWS はバックエンドであり、ストレージである。しかし SaaS の全ての特徴を、数学的な object と morphism で考えたときに、**記録の主座**はどこにあるか。それは **https://** という**境界面**である。ユーザーが触れるのはすべてこの境界のこちら側であり、背後にある計算・永続化は「向こう側」に隠れている。境界面をビルドするのが GitHub である。リポジトリ、PR、Issue、Actions — すべてが「公開された知」の境界を形作り、その境界に履歴として刻まれる。

位置ポテンシャルの中心は、この意味で GitHub である。公開した瞬間に public に記載され、バージョン情報と履歴が残り、核戦争があったとしても永久凍土に保管されうる知の金庫の、**相対的停留**が GitHub に現れている。

---

## 3. 事象の地平線 — 境界面に全てを記録する構造

ブラックホールは、事象の地平線（イベントホライズン）の**外側**から見たとき、そこを通過した情報を「境界面に記録したもの」として扱うことができる。中身の詳細は見えないが、境界には質量・角運動量・電荷などが刻まれ、情報は失われないという考え方（ブラックホール情報パラドックスを巡る議論）がある。

GitHub も同型である。**境界面に全ての情報を記録する**。コミット、ブランチ、タグ、リリース — すべてが境界上の履歴として残る。バックエンドが GCP であれ AWS であれ、ユーザーと世界が参照するのは「その境界に現れた公開情報」である。つまり GitHub は、知の金庫として、**境界面に全てを記録する事象の地平線**になっているのではないか。

---

## 4. 永久凍土に眠るリブート用の金庫

ノルウェー領**スヴァールバル諸島**などの永久凍土には、人類が絶滅に近い災害・事故・戦争を起こした場合にも**リブート**できるようにするための資産が眠っている。動植物の設計図、種子、そして GitHub データ（またはそのアーカイブ）もそこに含まれる。境界面に刻まれた知は、分散複製であると同時に、物理的に永久凍土に保管される「最後のバックアップ」として、文明の再起動のための種になっている。

| 保管物 | 役割 |
|--------|------|
| 種子 | 作物・生態の再開 |
| 動植物の設計図 | 生命の情報の復元 |
| GitHub データ（等） | ソフトウェア・知の境界面の復元 |

知の金庫は、オンラインの境界面であるとともに、オフラインの永久凍土にも存在する。スヴァールバルなどは、その**物理的な停留**なのである。

---

## 5. まとめ

| 概念 | 対応 |
|------|------|
| 知の金庫の相対的停留 | GitHub（v x.y.z 管理・全球同期配信） |
| 記録の主座 | https:// という境界面 |
| 境界面のビルダー | GitHub（リポジトリ・PR・履歴） |
| 比喩 | 事象の地平線 — 境界に全てが記録される構造 |
| 物理的な停留・リブート用 | スヴァールバル諸島等の永久凍土（種子・設計図・GitHub データ等） |

ソフトウェアの素晴らしさの本質は、Distributed Data Center と、その上に張られた**境界面**にある。その境界を組み立て・永続化し、全世界に同期させる装置として、GitHub は現代最高の知の金庫であり、同時に、境界面に全ての情報を記録する**イベントホライズン**として機能している。その知は、永久凍土にもバックアップされ、絶滅に近い事態のあとでもリブートできるように眠っている。
